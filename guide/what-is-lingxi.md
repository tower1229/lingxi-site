# 什么是灵犀

灵犀（LingXi）是一个 **Cursor 插件**，为开发者提供**持久记忆工作流**——让 AI 按你的方式做事。

## 痛点

在使用 Cursor 等 AI 编程工具时，你可能会遇到这些问题：

- **每次对话从零开始**：AI 不记得你之前的决策、偏好和踩过的坑
- **工程流程缺失**：AI 直接写代码，缺少需求分析、方案设计、代码审查等关键环节
- **上下文失控**：对话越长信息越杂，AI 的输出质量不断下降
- **团队经验无法传承**：每个人和 AI 的协作经验都封闭在自己的对话里

## 灵犀的解决方案

### 🧠 持久化记忆

灵犀通过 **/remember**、**/extract** 主动沉淀你在开发过程中的判断、偏好和经验教训，或在 **task / plan / build / review** 等环节中由工作流内置品味嗅探收集你的选择并写入，或在 **/init** 初始化项目时先生成候选并按你的明确选择可选写入，形成结构化的「记忆笔记」。记忆沉淀**不会自动执行**，仅由上述方式触发。在每一轮新对话中，灵犀会**自动检索并注入**最相关的记忆，让 AI 真正"了解"你的做事风格。

### 🔄 可伸缩工作流

从需求到交付的完整流程：

```
/task → /vet → /plan → /build → /review
```

流程按需组合、入口解耦。简单任务可直接 `/task` + `/build`，复杂任务可以走完整流程。你来决定何时使用哪条命令。

### 🛡️ 人工门控

AI 不会自作主张。关键决策——需求确认、记忆写入、方案选择——始终需要你点头。

### 🎯 上下文运营

通过语义 + 关键词双路径混合检索，灵犀每轮只注入 0–3 条最相关的记忆，避免信息过载，让模型聚焦关键信息。

## 设计理念

| 理念          | 含义                                                   |
| ------------- | ------------------------------------------------------ |
| **心有灵犀**  | 持久化记忆，让 AI 按你的方式做事                       |
| **AI Native** | 尊重 AI 能力，预留进化空间；关键决策以人为主、门控保障 |
| **称心如意**  | 降低认知负担，提供友好体验                             |

## 与 Cursor Rules / 其他方式的区别

灵犀提供**持久化记忆 + 结构化工作流**，与静态的 Cursor Rules、一次性 prompt 不同：侧重跨会话的「学习」与可解耦、按需推进的工作流（task / vet / plan / build / review）；记忆通过 **/remember**、**/extract** 按需沉淀，或由工作流（task/plan/build/review）中的品味嗅探在情境驱动时收集选择并写入，或在使用 **/init** 初始化时先给候选、再按你的选择可选写入，在新对话中按需注入，而不是固定写死的规则。

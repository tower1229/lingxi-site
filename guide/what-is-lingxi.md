# 什么是灵犀

灵犀（LingXi）是一个 **Cursor 插件**，为开发者提供**持久记忆工作流**——让 AI 按你的方式做事。

## 痛点

在使用 Cursor 等 AI 编程工具时，你可能会遇到这些问题：

- **每次对话从零开始**：AI 不记得你之前的决策、偏好和踩过的坑
- **工程流程缺失**：AI 直接写代码，缺少需求分析、方案设计、代码审查等关键环节
- **上下文失控**：对话越长信息越杂，AI 的输出质量不断下降
- **团队经验无法传承**：每个人和 AI 的协作经验都封闭在自己的对话里

## 灵犀的解决方案

### 🧠 持久化记忆

灵犀通过三种途径沉淀你在开发过程中的判断、偏好和经验教训，形成结构化的「记忆笔记」：**自动捕获**（自动会话提炼）、**手动捕获**（/remember、/init 等）、**品味嗅探**（在 task/plan/build/review 等工作流环节中情境驱动地收集你的选择并写入，详见 [品味嗅探](/guide/taste-sniffing)）。在每一轮新对话中，灵犀会**自动检索并注入**最相关的记忆，让 AI 真正"了解"你的做事风格。

### 🔄 自我迭代

灵犀具有**自我迭代**能力：运行期间会持续收集系统运行状态，并通过默认 24 小时的心跳间隔，对运行状态进行审计。在审计中发现的、仅涉及低风险的改进（例如记忆库的合并建议、索引补全等）会在后台自动应用，无需你额外操作；主会话不被打断，你只会收到一份简短简报。自我迭代是灵犀的全局特性，未来将逐步覆盖工作流、门控与更多子系统——让灵犀在长期使用中越用越稳、越用越懂你。

### 🔄 可伸缩工作流

从需求到交付的完整流程由 **task、vet、plan、build、review** 五个 **Skill** 驱动：

```
task → vet → plan → build → review
```

通过输入 `/task`、`/plan` 等或自然语言（如「执行 task」）**显式触发**；这些 Skill 仅支持手动或显式调用，不会根据语义自动加载。流程按需组合、入口解耦。简单任务可直接 task + build，复杂任务可以走完整流程。你来决定何时使用哪个 Skill。

### 🛡️ 人工门控

AI 不会自作主张。关键决策——需求确认、记忆写入、方案选择——始终需要你点头。

### 🎯 上下文运营

通过语义 + 关键词双路径混合检索，灵犀每轮只注入 0–3 条最相关的记忆，避免信息过载，让模型聚焦关键信息。

## 设计理念

| 理念          | 含义                                                   |
| ------------- | ------------------------------------------------------ |
| **心有灵犀**  | 持久化记忆，让 AI 按你的方式做事                       |
| **AI Native** | 尊重 AI 能力，预留进化空间；关键决策以人为主、门控保障 |
| **称心如意**  | 降低认知负担，提供友好体验                             |

## 与 Cursor Rules / 其他方式的区别

灵犀提供**持久化记忆 + 结构化工作流**，与静态的 Cursor Rules、一次性 prompt 不同：侧重跨会话的「学习」与可解耦、按需推进的工作流（task / vet / plan / build / review **Skills**）；记忆通过**自动捕获**（自动会话提炼）、**手动捕获**（/remember、/init）与**品味嗅探**（工作流 Skills 中情境驱动，各环节内容见 [品味嗅探](/guide/taste-sniffing)）三种途径沉淀，在新对话中按需注入；**自我迭代**作为全局特性在后台定期做诊断与自动改进，目前实现包含记忆改进，未来将扩展至全系统。
